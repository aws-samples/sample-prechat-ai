# 유스케이스와 참여자

## 누가 사용하나요?

| 역할 | 하는 일 |
|------|--------|
| **영업/마케팅 담당자** | 캠페인과 세션을 만들어 고객에게 전달하고, 상담 에이전트가 정리한 리포트로 미팅을 준비합니다. |
| **고객** | 링크를 받아 챗봇과 대화합니다. 양식을 채우는 대신 자연스럽게 요구사항을 전달합니다. |
| **운영 관리자** | 에이전트를 설정하고 캠페인 전체를 관리합니다. IT 담당자, CS 리더, PM 등이 맡습니다. |

{% hint style="info" %}
영업 담당자와 운영 관리자는 같은 사람이 겸할 수 있습니다. 소규모 팀이라면 한 명이 캠페인 생성부터 리포트 확인까지 모두 처리합니다.
{% endhint %}

## 대표 시나리오

### 시나리오 1. 개별 고객 사전상담

> 영업이 특정 고객을 지정해서 미팅 전 정보를 미리 수집하는 경우입니다.

{% stepper %}
{% step %}
### 아웃바운드 캠페인 만들기

"Q2 Enterprise Migration" 같은 이름으로 캠페인을 만들고, 상담 목적에 맞는 에이전트를 연결합니다.
{% endstep %}

{% step %}
### 고객별 세션 만들어 주기

고객 이름과 이메일을 입력하면 전용 URL과 PIN이 발급됩니다.
{% endstep %}

{% step %}
### 고객에게 상담 접속정보 알려주기

이메일이나 메신저로 URL과 PIN을 전달합니다. 고객은 별도 회원가입 없이 바로 접속합니다.
{% endstep %}

{% step %}
### 고객이 상담 에이전트와 대화

현재 상황, 원하는 것, 예산, 일정 등을 자연스럽게 이야기합니다. 폼을 채우는 부담 없이 대화로 진행됩니다.
{% endstep %}

{% step %}
### AI 리포트로 고객 미팅 준비

대화가 끝나면 BANT 요약과 미팅 플랜이 자동 생성됩니다. 영업팀은 이 리포트를 보고 본 미팅에 들어갑니다.
{% endstep %}
{% endstepper %}

### 시나리오 2. 세미나/이벤트 후속 접수

> 다수의 잠재 고객에게 상담 링크를 뿌려서 관심 있는 사람이 직접 신청하는 경우입니다.

{% stepper %}
{% step %}
### 인바운드 캠페인 만들기

캠페인 URL과 공통 PIN을 발급받습니다.
{% endstep %}

{% step %}
### 고객에게 상담 접속정보 알려주기

세미나 종료 슬라이드, 후속 이메일, QR 코드 등으로 "상담 신청은 여기서" 하라고 안내합니다.
{% endstep %}

{% step %}
### 고객이 상담 에이전트와 대화

PIN 입력 → 이름·연락처 입력 → 세션 자동 생성. 같은 연락처로 재방문하면 기존 대화가 이어집니다.
{% endstep %}

{% step %}
### 캠페인 대시보드에서 응답 현황 확인

세션 수, 상담 목적 분포, 만족도 점수를 실시간으로 모니터링합니다. 관심도가 높은 고객을 우선 팔로업합니다.
{% endstep %}
{% endstepper %}

### 시나리오 3. 기술 상담 채널 운영

> 파트너나 기존 고객이 기술 질문을 남기면 상담 에이전트가 1차 응대하고, 담당자가 후속 조치하는 경우

{% stepper %}
{% step %}
### 기술 상담 전용 캠페인을 생성

기술 문서 검색이 가능한 에이전트를 연결합니다. 고객 질문에 공식 문서 기반으로 답변합니다.
{% endstep %}

{% step %}
### 고객에게 상담 접속정보 알려주기

포털이나 슬랙 채널에 링크를 고정해 두면, 필요할 때 바로 접속할 수 있습니다.
{% endstep %}

{% step %}
### 상담 에이전트가 1차 응대

고객 질문에 즉시 답변하면서 대화 내용을 요약합니다. 담당자는 리포트를 보고 추가 지원이 필요한 건만 직접 대응합니다.
{% endstep %}
{% endstepper %}

## 어떤 캠페인 유형을 써야하나요?

핵심 기준은 **"고객을 내가 지정하느냐, 고객이 스스로 찾아오느냐"** 입니다.

고객을 이미 알고 있다면 아웃바운드를 쓰세요. 세션마다 고객 정보를 넣고 개별 링크를 보내면 됩니다. 

반대로 세미나 후속이나 마케팅 캠페인처럼 누가 올지 모르는 상황이라면 인바운드가 적합합니다. 링크로 관심 있는 고객이 접속하게 합니다.

| 상황 | 권장 유형 | 이유 |
|------|---------|------|
| 특정 고객 1:1 상담 | **아웃바운드** | 고객별 맞춤 세션으로 정보 품질이 높음 |
| 불특정 다수 접수 (세미나, 마케팅 캠페인 등) | **인바운드** | 링크 하나로 대량 접수, 운영 부담 최소 |
| 고객마다 다른 질문지·에이전트 필요 | **아웃바운드** | 세션별로 에이전트를 다르게 지정 가능 |
| 상시 운영 기술 상담 채널 | **인바운드** | 항상 열어 두고 필요할 때 접속하게 함 |

{% hint style="info" %}
한 계정에서 여러 캠페인을 동시에 운영할 수 있고, 아웃바운드와 인바운드를 혼합해도 됩니다. 처음엔 하나로 시작하고, 용도가 늘면 캠페인을 추가하세요.
{% endhint %}
