# 유스케이스와 참여자

## 참여자

| 역할 | 설명 | 인증 |
|------|------|------|
| **관리자 (Admin)** | 캠페인과 에이전트를 만들고 세션을 운영하는 ISV 고객사 담당자. CS 리더, 영업 담당자, PM 등이 맡습니다. | Cognito 이메일 + 비밀번호 |
| **고객 (Customer)** | 사전상담을 받는 엔드 유저. ISV 고객사의 잠재/기존 고객입니다. | 세션 또는 캠페인 PIN (6자리) |
| **솔루션 아키텍트 (SA)** | 에이전트의 프롬프트와 도구 구성을 설계하는 기술 전문가. 관리자 권한으로 작업합니다. | Cognito |

## 대표 시나리오

### 시나리오 1. 개별 맞춤 상담 (아웃바운드)

{% stepper %}
{% step %}
### 관리자가 "Q2 Enterprise Migration" 캠페인을 만든다

migration 전문 에이전트를 연결하고 캠페인을 활성화합니다.
{% endstep %}

{% step %}
### 고객사 A를 위한 세션을 생성한다

고객 이름, 이메일, 회사를 입력하면 세션 URL과 6자리 PIN이 발급됩니다.
{% endstep %}

{% step %}
### URL과 PIN을 이메일로 전달한다

고객이 URL에 접속하고 PIN을 입력합니다.
{% endstep %}

{% step %}
### 고객이 챗봇과 대화한다

현재 인프라, 마이그레이션 목표, 예산, 일정 등을 자연스럽게 공유합니다.
{% endstep %}

{% step %}
### AI가 BANT 요약과 미팅 플랜을 생성한다

영업팀은 세션이 끝난 뒤 리포트를 보고 본 미팅을 준비합니다.
{% endstep %}
{% endstepper %}

### 시나리오 2. 대규모 세미나 후속 접수 (인바운드)

{% stepper %}
{% step %}
### 관리자가 인바운드 캠페인을 만든다

캠페인 URL과 공통 PIN을 발급받습니다.
{% endstep %}

{% step %}
### URL을 세미나 참석자에게 공유한다

"상담 신청은 이 링크에서" 같은 형태로 안내합니다.
{% endstep %}

{% step %}
### 참석자가 직접 URL에 접근한다

PIN 입력 → 이름/전화번호 등 PII 입력 → 세션 자동 생성.
{% endstep %}

{% step %}
### 전화번호 기반 중복이 방지된다

같은 전화번호로 재진입하면 기존 세션을 복원합니다. 새 세션이 중복 생성되지 않습니다.
{% endstep %}

{% step %}
### 관리자는 쏟아지는 세션을 집계한다

캠페인 분석 대시보드에서 세션 수, 상담 목적 분포, CSAT 점수를 실시간으로 봅니다.
{% endstep %}
{% endstepper %}

### 시나리오 3. 파트너 기술 상담

{% stepper %}
{% step %}
### SA가 기술 상담 전용 에이전트를 설계한다

시스템 프롬프트에 기술 전문성을 주입하고, AWS Documentation MCP 도구를 활성화합니다.
{% endstep %}

{% step %}
### 관리자가 파트너 전용 캠페인을 만든다

에이전트를 연결하고 캠페인을 공개합니다.
{% endstep %}

{% step %}
### 파트너가 기술 질의를 입력하면 AI가 공식 문서로 답한다

AWS Documentation MCP가 실시간 문서를 검색해 정확한 기술 정보를 제공합니다.
{% endstep %}
{% endstepper %}

## 어떤 유형을 골라야 하나

| 상황 | 권장 유형 |
|------|---------|
| 특정 고객 1:1 상담, 민감 정보 포함 | **아웃바운드** |
| 세미나, 마케팅 캠페인 후속 접수 | **인바운드** |
| 내부 파트너/직원 대상 Q&A | **인바운드** |
| 고객마다 질문지와 에이전트를 다르게 구성 | **아웃바운드** (세션별 에이전트 오버라이드 가능) |

{% hint style="info" %}
한 관리자 계정에서 여러 캠페인을 동시에 운영할 수 있으며, 유형도 혼합 가능합니다.
{% endhint %}
